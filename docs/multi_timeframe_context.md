# 多周期上下文

## 用途

主周期是交易训练与事件记录所使用的 K 线周期。例如用户以 `1m`
回放时，开仓、平仓、事件标签与 `bar_index` 都属于这组 `1m` 数据。

多周期上下文面板显示更高周期的只读市场状态。`1m` 主周期默认显示
`5m` 与 `15m`，并可勾选 `1h`。它用于复盘时观察更长周期背景，不是
交易信号，也不会自动触发任何交易行为。

## 同步规则

高周期同步依据当前主周期 cursor 所在 K 线的 `open_time_bjt`，不依据
主周期 `bar_index`。

面板可以定位时间上包含 cursor 的高周期 K 线，但状态统计只使用在该
cursor 时间之前已经完成的高周期 K 线。这样做是为了避免历史回放中
使用尚未闭合的大周期 K 线最终收盘价、高点或低点。

每个周期显示：

- 同步状态
- 可安全使用的 HTF bar 时间与收盘价
- 过去 20 根收益和实现波动率
- 趋势状态与波动状态

历史不足 20 根时，面板明确显示样本不足，不补造历史。

## 数据边界

- 高周期 `bar_index` 不写入主周期交易事件。
- 高周期状态不写入交易样本、研究 schema 或会话交易事务。
- 高周期加载使用与主周期相同的品种和日期范围，优先读取缓存。
- 高周期数据加载失败时，只显示失败信息并记录日志；主周期 K 线、
  cursor、session 和交易记录不受影响。
- 主周期品种、周期或日期变化后，旧高周期上下文立即标记为失效，
  等待主周期重新加载。
