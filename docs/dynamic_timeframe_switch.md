# 动态周期切换

## 显示周期与样本周期

`display_interval` 是主图当前展示和播放的周期。用户切换周期后，主图
自动加载新的 K 线，并保持在原市场时间附近。

`sample_interval` 是当前 session 已记录交易样本所使用的周期。交易、
事件窗口和 `bar_index` 都必须继续服从该周期。

如果 session 已经包含 `1m` 交易或事件，主图仍可临时切换到 `5m`
观察走势，但此时开仓、平仓按钮被禁用，旧 `1m` marker 不会画到
`5m` 图表。用户要在 `5m` 上记录交易，需要新建 `5m` session。

## 时间锚点

周期切换按当前 cursor 对应的 `open_time_bjt` 定位，不按 cursor 数字
定位。例如，`1m` 图上 `10:37` 的 cursor 切换到 `5m` 后，会落到覆盖
该时间的 `10:35` K 线。

可视区也按市场时间跨度恢复。原图若显示约一小时，切换到较高周期后
仍显示约一小时附近的数据，而不是机械保留原来的 K 线根数。

## 播放与失败处理

- 暂停状态切换周期后，定位完成仍保持暂停。
- 播放状态切换周期后，旧图立即停止推进；新周期加载并定位成功后继续播放。
- 加载失败时，旧图表数据、旧 cursor 和已有交易样本保留，播放保持暂停。
- 主周期加载过程中再次选择周期时，只排队最后一次选择，避免并发覆盖。

## 产品边界

动态周期切换用于历史回放查看与手工训练。它不接实盘，不产生交易
信号，不改变研究 schema，也不允许将不同周期的 `bar_index` 静默写入
同一组交易样本。
